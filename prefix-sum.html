Concept

The Big Idea

Asked "what is the sum of elements from index L to R?" naively costs O(n) per query. Build a prefix-sum array once in O(n), then answer every query in O(1) with a single subtraction.

prefix[i] = arr[0] + arr[1] + ... + arr[i]

Then: sum(L, R) = prefix[R] − prefix[L−1]  (prefix[-1] = 0)

Build: O(n)
Each Query: O(1)
Without prefix: O(n) per query
#include <bits/stdc++.h>
using namespace std;
int main() {
    vector<int> arr = {3,1,4,1,5,9,2,6};
    int n = arr.size();
    vector<int> pre(n);
    pre[0] = arr[0];
    for(int i=1; i<n; i++)
        pre[i] = pre[i-1] + arr[i];
    // sum(2,5) = pre[5] - pre[1] = 20 - 4 = 16? wait:
    // arr[2..5] = 4+1+5+9 = 19
    // pre[5]=23, pre[1]=4 → 23-4 = 19 ✓
    int ans = pre[5] - pre[1];  // L=2, so pre[L-1]=pre[1]
    cout << ans;  // 19
}
Practice Problem

Range Sum Queries

Given arr = [3, 1, 4, 1, 5, 9, 2, 6], answer queries: sum(0,3), sum(2,5), sum(4,7).

Without prefix sum: Each query scans up to n elements → slow for many queries.
With prefix sum: Build once, then each answer is just pre[R] - pre[L-1].

// pre = [3, 4, 8, 9, 14, 23, 25, 31]
// sum(0,3) = pre[3]        = 9    → 3+1+4+1
// sum(2,5) = pre[5]-pre[1] = 23-4 = 19 → 4+1+5+9
// sum(4,7) = pre[7]-pre[3] = 31-9 = 22 → 5+9+2+6
