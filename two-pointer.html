Concept

The Idea

Two pointers is a technique where you use two indices that move through an array — either toward each other (converging) or at different speeds (slow + fast). This turns many O(n²) brute-force problems into O(n).

Common patterns:

  • Slow + Fast pointer — slow writes unique elements, fast scans ahead (used in: remove duplicates, partition, Dutch flag)
  • Left + Right pointer — converge from both ends on sorted data (used in: two-sum, container with most water)
Brute Force: O(n²)
Two Pointer: O(n)
// Remove duplicates from sorted array in-place
int removeDuplicates(vector<int>& arr) {
    if(arr.empty()) return 0;
    int slow = 0;  // slow: last unique position
    for(int fast = 1; fast < arr.size(); fast++) {
        if(arr[fast] != arr[slow]) { // found new unique
            slow++;
            arr[slow] = arr[fast]; // write it
        }
        // else: duplicate → fast moves, slow stays
    }
    return slow + 1; // count of unique elements
}
Practice Problem

Remove Duplicates from Sorted Array

Given sorted array arr = [1, 1, 2, 2, 2, 3, 4, 4, 5], remove duplicates in-place and return count of unique elements.

Key insight: slow marks where to write the next unique value. fast scans every element. When fast finds something different from slow's value → write it at slow+1 and advance slow.

Result: [1, 2, 3, 4, 5, _, _, _, _] — 5 unique elements. ✅
