The Technique

A sliding window maintains a contiguous subarray of fixed (or variable) size and "slides" it one step at a time. Instead of recomputing the whole window, we just add the new element and remove the leftmost element.

This turns O(n×k) brute force into O(n).

  • Fixed window: window size k is constant — subtract left, add right
  • Variable window: expand right until condition breaks, then shrink from left
Brute Force: O(n·k)
Sliding Window: O(n)
// Max sum of any subarray of size k
int maxSumK(vector<int>& arr, int k) {
    int n = arr.size(), winSum = 0;
    for(int i=0; i<k; i++) winSum += arr[i]; // first window
    int maxSum = winSum;
    for(int i=k; i<n; i++) {
        winSum += arr[i];         // add new right
        winSum -= arr[i-k];       // remove old left
        maxSum = max(maxSum, winSum);
    }
    return maxSum;
}
Practice Problem

Maximum Sum Subarray of Size K

Given arr = [2, 1, 5, 1, 3, 2] and k = 3, find the maximum sum of any contiguous subarray of size 3.

Windows: [2,1,5]=8 → [1,5,1]=7 → [5,1,3]=9 → [1,3,2]=6  →  Answer: 9

// Trace for arr=[2,1,5,1,3,2], k=3:
// i=0..2: sum = 2+1+5 = 8, max = 8
// i=3:    sum = 8+1-2 = 7, max = 8
// i=4:    sum = 7+3-1 = 9, max = 9  ✓
// i=5:    sum = 9+2-5 = 6, max = 9
