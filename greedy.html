Concept

What is Greedy?

A greedy algorithm makes the best possible choice at each step without reconsidering past choices. It hopes that locally optimal decisions lead to a globally optimal solution.

When does greedy work? When the problem has the greedy choice property — local optimal → global optimal. Classic examples:

  • Coin change (standard denominations)
  • Activity selection / interval scheduling
  • Huffman encoding
  • Kruskal's MST (minimum spanning tree)

⚠️ Greedy doesn't always work! For coin change with arbitrary denominations, you need DP.

// Coin Change (Greedy — works for standard coins)
int minCoins(int amount) {
    vector<int> coins = {500,200,100,50,20,10,5,2,1};
    int count = 0;
    for(int c : coins) {
        count += amount / c;   // use as many as possible
        amount %= c;           // remainder
    }
    return count;
}
Practice Problem

Minimum Coins to Make Change

Given coins = [1, 2, 5, 10, 20, 50, 100, 200] (Indian paisa-style), find the minimum coins to make amount = 93.

Greedy: 50+20+20+2+1 = 5 coins ✅

// amount=93
// 93 / 50 = 1 coin → rem = 43
// 43 / 20 = 2 coins → rem = 3
// 3  / 2  = 1 coin  → rem = 1
// 1  / 1  = 1 coin  → rem = 0
// Total = 5 coins ✅
